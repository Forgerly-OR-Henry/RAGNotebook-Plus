- 项目：PKM-2026 个人知识库复用研究
- 标签：复盘, RAGNotebook, 用户体验

## 最有用的部分

上传多种文件后，林澈可以用同一个问题同时触达 PDF 讲义、DOCX 纪要和 Markdown 日志。

知识库详情页能查看原文件和片段，降低了直接复制模型回答的风险。

笔记导出为 Markdown 后，适合继续放入论文草稿或个人博客。

## 仍然别扭的部分

PPTX 中的短句被切片后语境较弱，最好在幻灯片里写完整备注或补充说明。

同一主题资料过多时，用户需要先整理问题，否则检索结果会显得杂。

如果文件名太抽象，后续判断来源会变慢，所以文件名应该包含主题和时间。

## 给后续自己的规则

每个文件至少写一句“为什么保存”。

重要资料同时保留原文和自己的解释。

组会前先检索问题，再决定 PPT 结构，不再先做幻灯片。

## 复用提示

这份材料保留了保存时的任务场景，后续可用于组会汇报、阶段简报和论文草稿中的证据链整理。
