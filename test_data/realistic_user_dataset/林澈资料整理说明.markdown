# 资料整理说明：林澈的文件命名和保存规则

- 日期：2026-05-12
- 类型：资料整理说明
- 人物：林澈，信息管理方向研二学生
- 项目：PKM-2026 个人知识库复用研究
- 标签：资料组织, 文件命名, 工作流

## 命名规则

文件名先写主题，再写用途，最后写时间或阶段。例如 项目日志-个人知识库复用研究.md 比 log.md 更容易被回忆。

如果文件来自正式讨论，用 meeting、brief、feedback 等词保留协作语境。

如果文件是个人过程记录，用 log、capture、fragments 等词保留未完成状态。

## 保存规则

PDF 保存正式资料和讲义，DOCX 保存会议纪要和可发给导师的文档，PPTX 保存组会表达，Markdown 保存个人解释和索引。

每次上传后补一句“为什么留下它”。这句话往往能在两周后帮助自己重新进入问题。

不要追求一次整理到位，先保证资料能被找到，再逐步补结构。

## 复盘规则

每周五检查哪些资料被重复使用。重复使用的材料应提升为主题索引中的证据。

每次组会后更新导师反馈和问题清单，避免反馈散落在聊天记录里。

当某个主题超过 8 个文件时，建立主题索引，而不是继续加子文件夹。

## 复用提示

这份材料保留了保存时的任务场景，后续可用于组会汇报、阶段简报和论文草稿中的证据链整理。
