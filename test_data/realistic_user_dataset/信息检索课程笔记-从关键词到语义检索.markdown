## 课堂主线

传统信息检索强调倒排索引、词项匹配和排序函数。它对明确关键词很有效，但对“我记得意思却忘了词”的情况支持不足。

语义检索通过向量表示缓解词汇不匹配问题，但它也带来可解释性问题：用户需要知道为什么某个片段被命中。

在个人知识库中，检索不是一次性找答案，而是帮助用户回到某个学习或写作场景。

## 与 PKM-2026 的连接

林澈的研究可以把课程里的 precision、recall 转译成更贴近人的指标：能否找回关键材料、能否判断来源可信、能否继续写作。

资料类型对检索有影响。PDF 常是正式来源，Markdown 常是过程记录，DOCX 常是协作或导师反馈，PPTX 常是凝练后的表达。

如果系统能显示文件类型、上传时间和片段位置，用户对结果的信任会更高。

## 课后问题

BM25 与向量检索在个人知识库中如何互补？

如果用户的问题很长，系统是否应该先改写查询？改写后的查询是否还保留用户意图？

排序结果是否应考虑资料的新旧、文件夹、用户手动置顶等信号？

## 可复用片段

个人知识库的检索目标是恢复上下文，而不只是返回相关文本。

语义检索提高了召回可能性，但来源可审计性决定了用户是否敢把结果写进正式文档。

资料组织方式和检索界面会共同影响研究者对知识的复用方式。
